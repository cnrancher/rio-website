为Kubernetes打造极简MicroPaaS平台

Rio可在任何 Kubernetes 集群中构建、测试、部署、伸缩和版本无状态应用。



什么是MicroPaaS？


MicroPaaS是一个轻量级框架，可以弥补IT基础设施和开发人员之间的差距。模块化架构让Rio无需接管集群，亦能获得从Pipeline到操作系统的完整体验。多租户设计，像Rio这样的MicroPaaS平台可以在任何底层上的任何Kubernetes集群上运行……甚至包括笔记本电脑。

Rio是如何建设的？


Rio创建了关于Istio和Knative的极简体验，并提供了一种管理应用程序各个方面的简便方式。


Rio包括：

  • 自动 DNS和HTTPS
  • HTTP 负载均衡
  • HTTP 路由
  • HTTP 指标
  • 自动扩缩容
  • 金丝雀部署
  • Git触发的部署

为什么选择Rio？



创建应用程序，而非平台

"Rio可安装到任何Kubernetes集群中，并处理常见服务的所有连接。例如服务网格的Istio，用于创建事件函数的Knative以及用于监控的Prometheus。 通过一些简单的命令，Rio可以实现基于Git的持续交付；帮助优化应用程序的路由；提供金丝雀部署、AB测试及蓝绿部署；监控服务网格内的HTTP流量；提供智能自动伸缩等。"


简化发布Pipeline

Rio帮助您为用户提供应用程序的最佳发布体验。

  • 当您提交对存储库的更改时，Rio可帮您自动构建和部署新版本的应用程序
  • 在推出应用程序的新版本时，Rio可了解当前负载，并利用Prometheus第4层和第7层自动伸缩，获取最佳性能。


默认的最高安全级别

使用Rio时，您的应用程序可以通过最安全的网络配置启动。

  • 您的工作负载从LetsEncrypt接收DNS记录和通配符证书。
  • Rio整合了Kubernetes RBAC，以获得最大的安全性。
  • 您的应用程序将自动包含资源约束，达到集群的最佳性能体验。
